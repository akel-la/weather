# ОПИСАНИЕ ЗАДАНИЯ:

Суть
Написать скрипт, который принимает название города и выводит текущую погоду.

Что должно быть:

(1) Интеграция с OpenWeatherMap API (или любым бесплатным аналогом) для получения актуальных данных о погоде
(2) Ключ API хранится в файле .env, не зашит в коде и не публикуется в репозитории
(3) Скрипт запускается из командной строки: python weather.py Moscow (название города передаётся аргументом)
(4) В консоль выводится: текущая температура в градусах Цельсия и текстовое описание погоды (например, "ясно", "облачно")
(5) Если город не найден или API вернул ошибку — вывести понятное сообщение об ошибке, программа не должна падать

Ссылка на публичный GitHub-репозиторий. В корне репозитория должны быть: weather.py, .env.example (с примером формата), README.md с командой запуска и краткой инструкцией по установке зависимостей.

# ОБЩЕЕ:

(1) У OpenWeatherMap API есть 2 API - 3.0 и 2.5. Версия 2.5 не требует привязать банковскую карту, поэтому использовал эту версию.

(2) Не стал разделять программу на большое количество отдельных файлов (settings.py, cli.py, weather.py, ...) так как программа маленькая и для скрипта важно, чтобы он работал быстро, а импорты тратят время. Если сделать программу больше - тогда уже можно разносить по файлам.

(3) Добавил несколько флагов для командной строки, без этого программа была бы слишком простой - даже на тестовое для Джуна слишком просто.

(4) Есть настройки языка для того, на каком языке будет получена информация по API, но не стал реализовывать разные языки для сообщений самой программы (чтобы не загромождать программу кучей строк на разных языках). Из-за этого при переключении на другой язык получается смесь разных язков.

(5) Сделал так, чтобы можно было легко добавить новый сервис со своим API или даже несколькими его версиями, причем у каждой может быть свой API ключ, но это за счет усложения кода (к примеру, используя \__init_sublcass__).

(6) Использую raise SystemExit, чтобы программа завершилась (если что-то идет не так), при этом завершение не мгновенное (к примеру, как при os._exit или sys.exit), т.е. программа сначала сделает то, что нужно перед завершением, а в вызове (блок try в if \__name__ == "__main__") мы перехватываем SystemExit и выводим сообщение об ошибке без трассировки стека и прочего, что не нужно пользователям программы. Если потребуется запускать программу в режиме разработки - то просто комментируем блок except SystemExit as e, и получим подробности.

(7) API 2.5, если есть запрос на город в стране, в которой его нет или если город и/или страна не существуют, может вести себя по разному - возвращать 404 или возвращать 200 и данные о городе, который находится не в той стране, которую указал пользователь, поэтому такая немного сложная логика в get_weather.

(8) Зарегестрируетесь и получите реальный ключ в OpenWeatherMap - иногда нужно немного подождать, пока их сервис введет ключ в работу, первые несколько минут может не рабоать.

# ПРИМЕРЫ РАБОТЫ:

## КОРРЕКТНАЯ РАБОТА:



python3 weather.py London

В городе London сейчас 15.65°C, пасмурно.



python3 weather.py London --lang en

В городе London сейчас 15.7°C, overcast clouds.



python3 weather.py London --units imperial

В городе London сейчас 60.17°F, пасмурно.

Сложный случай - города, которые называются одинаково, но находятся в разных странах:
(Город Виктория - есть в Канаде (CA) и на Сейшельских Островах (SC)

python3 weather.py Виктория
В городе Виктория сейчас 10.33°C, небольшая облачность.

python3 weather.py Виктория --country CA
В городе Виктория сейчас 10.4°C, небольшая облачность.

python3 weather.py Виктория --country SC
В городе Виктория сейчас 26.59°C, небольшая облачность.
